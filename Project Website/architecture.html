Architecture

What are the components of our application?

Architectural Diagram

Work allocation between our team and team 2

Our team is working on the following components:

  • User Interface
  • Graph Engine
  • Graph Library
  • Set Engine (with Team 2)
The other components are done by Team 2.

Architectural Components

User Interface: Toolbar and canvas which users use to see the visualised data.

Graph Engine: Uses the standard code for a type of diagram and accesses the data file to create a visualisation.

Graph Library: The code used to generate a diagram of the following types: line chart, bar chart, Venn diagram, pie chart and donut chart.

Set Engine: The back end code to merge and query different sets of data, done both by our team and Team 2.

Set Definition Library: The pre-defined sets of data created by Team 2 which our application links to to get data.

Data Sources: The data from the SIMS school data management application.
